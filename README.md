# ผู้ช่วยสื่ออัจฉริยะ (ปลั๊กอินอ่านภาพจากมือถือ)

> **หมายเหตุ:** ไฟล์นี้แปลจากภาษาจีนเป็นภาษาไทยโดย AI ด้วยอนุญาตจากผู้พัฒนาต้นฉบับ (kencuo / ctrl)  
> เครดิตและสิทธิ์ทั้งหมดเป็นของผู้พัฒนาเดิม

SillyTavern Extension: ประมวลผลรูปภาพและเอกสารข้อความแบบรวม (บีบอัด/บันทึก/ส่งไปแชท) พร้อมเปิดเผย API ส่วนกลางให้สคริปต์อื่นเรียกใช้

## ฟีเจอร์

- ประมวลผลรูปภาพ
  - รองรับรูปแบบรูปภาพทั่วไป (jpg/jpeg/png/gif/webp/bmp)
  - บีบอัดแบบเลือกได้: ย่อรูปให้ได้ขนาดด้านยาวสูงสุดแล้วแปลงเป็น JPEG
  - บันทึกไปยังไดเรกทอรีรูปภาพของผู้ใช้ผ่าน `saveBase64AsFile` ในตัว SillyTavern (โฟลเดอร์ย่อยเริ่มต้น: `phone/`)
- ประมวลผลเอกสาร
  - รองรับไฟล์ข้อความ txt/json/md/csv/html/xml/js/css/rtf
  - อ่านเป็นข้อความ UTF-8; จัดรูปแบบ json, ตัดแสดงตัวอย่าง csv
  - เลือกใช้ "AI อ่านเอกสาร": ส่งเนื้อหาเป็นข้อความผู้ใช้ไปยังแชท
- แผงการตั้งค่า
  - เพิ่มแผงพับได้ในหน้าตั้งค่า extension สำหรับคอนฟิกสวิตช์/คุณภาพรูปภาพ/ขนาดสูงสุด/ขนาดไฟล์ และอื่นๆ

## การติดตั้ง

1. วางไฟล์ extension นี้ในไดเรกทอรี extension ของ SillyTavern (ระดับเดียวกับ extension อื่น)
2. ไปที่ SillyTavern → Extensions (ส่วนขยาย) แล้วเปิดใช้งาน/รีเฟรช extension
3. ค้นหาแผง "ปลั๊กอินอ่านภาพ byctrl" ในหน้าตั้งค่า extension เพื่อคอนฟิก

> หมายเหตุ: Extension นี้ใช้ relative path import ของโมดูลภายใน SillyTavern frontend (เช่น `../../../../script.js`, `../../../utils.js`)  
> ถ้าวางไฟล์ในระดับไดเรกทอรีที่ต่างจากที่คาดไว้ อาจทำให้ import ล้มเหลว กรุณาตรวจสอบให้อยู่ในโครงสร้างไดเรกทอรี extension มาตรฐาน

## วิธีใช้งาน

### 1) เรียกใช้เป็น API ส่วนกลางจากสคริปต์อื่น

หลังจาก extension โหลดแล้ว ฟังก์ชันต่อไปนี้จะถูกเปิดเผยบน `window`:

- `window.__processFileByPlugin(file, options?)`
  - ระบุและประมวลผลรูปภาพ/เอกสารอัตโนมัติ
- `window.__uploadImageByPlugin(file, options?)`
  - ประมวลผลรูปภาพเดียว
- `window.__uploadMultipleImagesByPlugin(files, options?)`
  - ประมวลผลหลายรูปพร้อมกัน คืนค่า `results` และ `errors`
- `window.__processDocumentByPlugin(file, options?)`
  - ประมวลผลเอกสาร; เริ่มต้นจะส่งไปแชท (ปิดได้ด้วย `options.sendToChat=false`)
- `window.__isDocumentFile(file)`
  - ตรวจสอบว่าเป็นประเภทเอกสารที่รองรับหรือไม่
- `window.__getSupportedFileTypes()`
  - ดึงรายการ MIME และนามสกุลที่รองรับ

`options` (ตอนประมวลผลรูปภาพ/เอกสาร):

- `sendToChat` (เฉพาะเอกสาร): ใส่ `false` เพื่อปิดการส่งไปแชทอัตโนมัติ

### 2) Bridge เนื้อหาเอกสาร (วิธี Slash Command)

Extension ยังเปิดเผย:

- `window.smartMediaAssistant.processText(text, options?)`

ฟังก์ชันนี้จะพยายามส่งผ่าน `executeSlashCommandsWithOptions` หรือ `triggerSlash`:

- `/send <เนื้อหา> | /trigger`

ใช้สำหรับเรียกกระบวนการสร้าง/สรุปที่ตามมา

## คำอธิบายผลลัพธ์

### ค่าที่คืนจากการประมวลผลรูปภาพ

เมื่อประมวลผลสำเร็จจะคืนค่า:

- `success: true`
- `url`: URL ที่เข้าถึงได้จาก `saveBase64AsFile`
- `metadata`: ชื่อไฟล์ต้นฉบับ/หลังประมวลผล, ขนาด, เวลา และอื่นๆ

path การบันทึกเริ่มต้น (แนวคิด):

- `.../user/images/phone/<uniqueId>.jpg`

### ค่าที่คืนจากการประมวลผลเอกสาร

เมื่อประมวลผลสำเร็จจะคืนค่า:

- `success: true`
- `content`: เนื้อหาข้อความหลังประมวลผล
- `metadata`: ชื่อไฟล์/ประเภท/ขนาด/ความยาว/เวลา และอื่นๆ

## ตัวเลือกการตั้งค่า

ตั้งค่าได้ใน extension settings:

- เปิดใช้งานประมวลผลรูปภาพ / เปิดใช้งานประมวลผลเอกสาร
- คุณภาพรูปภาพ (10-100)
- ขนาดสูงสุดของรูปภาพ (512-4096)
- จำกัดขนาดไฟล์ (MB)
- เปิดใช้งาน AI อ่านเอกสาร (ส่งไปแชท)
- แสดงข้อมูลการประมวลผล (toastr notification)
- บันทึก Debug (console)

## ข้อจำกัดที่ทราบ / ข้อควรระวัง

- รูปภาพจะถูกแปลงเป็น JPEG ทั้งหมด: พื้นหลังโปร่งใส (PNG/WebP) จะสูญเสียความโปร่งใส
- การส่งเอกสารไปแชทใช้ `addOneMessage` (window/parent/top) เป็นลำดับแรก; ถ้าไม่พร้อมใช้งาน จะลองใช้ slash `/send ... | /trigger` (ขึ้นอยู่กับ `slash-commands.js` หรือ `triggerSlash`)
- Extension นี้ขึ้นอยู่กับโมดูลภายใน SillyTavern frontend (`getContext` / `saveBase64AsFile` ฯลฯ); ถ้าระดับไดเรกทอรีไม่ตรงทำให้ import ล้มเหลว จะ error และทำงานไม่ได้

## การพัฒนา / แก้ไขปัญหา

- เปิด "บันทึก Debug" แล้วดู log ที่มีคำนำหน้า `[Smart Media Assistant]` ใน Browser DevTools Console
- ถ้าพบ import error ให้ตรวจสอบระดับไดเรกทอรีที่วาง extension ว่าตรงกับ relative path หรือไม่
